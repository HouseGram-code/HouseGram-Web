# Инструкция по настройке Firebase Storage и запуску HouseGram

## 1. Настройка Firebase Storage Rules

Для работы загрузки стикеров необходимо настроить правила Firebase Storage:

### Способ 1: Через Firebase Console
1. Откройте [Firebase Console](https://console.firebase.google.com/)
2. Выберите ваш проект: `ai-studio-applet-webapp-7235b`
3. Перейдите в **Storage** → **Rules**
4. Скопируйте содержимое файла `storage.rules` и вставьте его
5. Нажмите **Publish**

### Способ 2: Через Firebase CLI
```bash
# Установите Firebase CLI если ещё не установлен
npm install -g firebase-tools

# Войдите в Firebase
firebase login

# Инициализируйте проект (если ещё не инициализирован)
firebase init storage

# Разверните правила Storage
firebase deploy --only storage
```

## 2. Запуск сервера разработки

```bash
# Установите зависимости (если ещё не установлены)
npm install

# Запустите сервер разработки
npm run dev
```

Сервер будет доступен по адресу: http://localhost:3000

## 3. Продакшен сборка

```bash
# Соберите оптимизированную версию
npm run build

# Запустите продакшен сервер
npm start
```

## 4. Проверка работы стикеров

После настройки Firebase Storage:

1. Откройте приложение в браузере
2. Войдите в систему (если требуется)
3. Откройте любой чат
4. Нажмите на иконку стикера (🙂 → Стикеры)
5. Нажмите кнопку "+" для загрузки своего стикера
6. Выберите изображение и введите название
7. Нажмите "Создать"

## 5. Возможные ошибки и решения

### Ошибка "Нет доступа для загрузки"
- Проверьте, что правила Storage правильно настроены (см. шаг 1)
- Убедитесь, что пользователь авторизован

### Ошибка "Неверный формат изображения"
- Попробуйте другое изображение
- Убедитесь, что изображение в формате PNG или JPG

### Стикеры не сохраняются
- Проверьте localStorage в браузере (F12 → Application → Local Storage)
- Должен быть ключ `housegram_stickers`
