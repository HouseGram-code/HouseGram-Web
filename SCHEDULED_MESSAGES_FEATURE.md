# 📅 Запланированные сообщения в HouseGram

## Обзор функции

Новая функция запланированных сообщений позволяет пользователям отправлять сообщения в определенное время в будущем, как в Telegram. Это полезно для:

- Отправки поздравлений в нужное время
- Напоминаний о важных событиях
- Сообщений в разных часовых поясах
- Автоматизации рутинных сообщений

## 🎯 Как использовать

### Планирование сообщения:
1. Напишите текст сообщения в поле ввода
2. **Удерживайте** кнопку отправки (📤) в течение 0.5 секунды
3. Появится модальное окно выбора времени
4. Выберите дату и время или используйте быстрые варианты
5. Нажмите "Запланировать"

### Быстрые варианты:
- **Через 1 час** - отправка через час от текущего времени
- **Через 3 часа** - отправка через 3 часа
- **Завтра в 9:00** - отправка завтра утром
- **Завтра в 18:00** - отправка завтра вечером

### Управление запланированными сообщениями:
1. Перейдите в **Настройки** → **Запланированные сообщения**
2. Просмотрите все запланированные сообщения
3. Отправьте сообщение немедленно или удалите его

## 🛠 Техническая реализация

### Компоненты:

#### 1. `ScheduleMessageModal.tsx`
Модальное окно для выбора времени отправки:
- Быстрые варианты времени
- Календарь для выбора даты
- Поле времени
- Предварительный просмотр
- Валидация (время должно быть в будущем)

#### 2. `ScheduledMessagesView.tsx`
Страница управления запланированными сообщениями:
- Список всех запланированных сообщений
- Информация о времени отправки
- Кнопки "Отправить сейчас" и "Удалить"
- Статус сообщений (запланировано/отправлено/ошибка)

#### 3. Обновленный `ChatInput.tsx`
- Поддержка долгого нажатия на кнопку отправки
- Визуальная индикация долгого нажатия
- Всплывающая подсказка
- Анимации и обратная связь

### База данных:

#### Коллекция `scheduledMessages`:
```typescript
interface ScheduledMessage {
  id: string;
  text: string;              // Текст сообщения
  scheduledFor: Date;        // Время отправки
  chatId: string;           // ID чата
  senderId: string;         // ID отправителя
  replyTo?: string;         // ID сообщения для ответа
  createdAt: Date;          // Время создания
  status: 'scheduled' | 'sent' | 'failed';
  sentAt?: Date;            // Время фактической отправки
  error?: string;           // Ошибка (если есть)
}
```

### API Endpoint:

#### `POST /api/scheduled-messages`
Обрабатывает запланированные сообщения:
- Находит сообщения, время которых наступило
- Отправляет их в соответствующие чаты
- Обновляет статус на 'sent' или 'failed'
- Возвращает статистику обработки

#### `GET /api/scheduled-messages`
Возвращает статистику:
- Общее количество запланированных сообщений
- Количество просроченных сообщений
- Временная метка

## 🔄 Автоматическая отправка

### Серверная логика:
Для автоматической отправки сообщений нужно настроить cron job или webhook, который будет вызывать API каждую минуту:

```bash
# Cron job (каждую минуту)
* * * * * curl -X POST https://your-domain.com/api/scheduled-messages
```

### Vercel Cron Jobs:
```json
{
  "crons": [
    {
      "path": "/api/scheduled-messages",
      "schedule": "* * * * *"
    }
  ]
}
```

### Альтернативы:
- **GitHub Actions** с расписанием
- **Внешние сервисы** (cron-job.org, EasyCron)
- **Serverless функции** с таймерами
