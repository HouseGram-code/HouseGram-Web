# Исправления: Сообщения боту и статус "в сети"

## Дата: 9 апреля 2026

## Исправленные проблемы

### 1. ✅ Сообщения боту не отображаются

**Проблема**: Когда пользователь отправлял сообщение боту (test_bot), его собственное сообщение не появлялось в чате.

**Причина**: В `ChatContext.tsx` строка 313 содержала `return`, который блокировал подписку на сообщения из Firebase для чата с ботом:

```typescript
// БЫЛО (неправильно):
if (activeChatId === 'saved_messages' || activeChatId === 'test_bot') return;
```

Это означало, что сообщения не загружались из Firebase для бота.

**Решение**: Убрали `test_bot` из условия, теперь подписка работает для всех чатов кроме `saved_messages`:

```typescript
// СТАЛО (правильно):
if (activeChatId === 'saved_messages') return;
```

Теперь:
- ✅ Твои сообщения боту сразу появляются в чате
- ✅ Ответы бота также отображаются
- ✅ Вся история сообщений сохраняется в Firebase

### 2. ✅ Пользователи показываются "в сети" когда вышли

**Проблема**: Пользователи продолжали отображаться как "в сети" даже после того, как закрыли приложение или вышли.

**Причина**: Логика проверки статуса не учитывала время последней активности. Если в базе данных стоял статус `online`, он показывался независимо от того, когда пользователь был активен последний раз.

**Решение**: Добавили проверку времени последней активности:

```typescript
// Проверяем статус И время последней активности
const isOnline = userData.status === 'online';
let isRecentlyActive = false;

if (userData.lastSeen) {
  const lastSeenDate = userData.lastSeen.toDate ? userData.lastSeen.toDate() : new Date(userData.lastSeen);
  const now = new Date();
  const diffMinutes = Math.floor((now.getTime() - lastSeenDate.getTime()) / 60000);
  
  // Считаем пользователя онлайн если он был активен менее 2 минут назад
  isRecentlyActive = diffMinutes < 2;
  
  // Вычисляем текст статуса
  if (diffMinutes < 1) {
    statusOffline = 'был(а) только что';
  } else if (diffMinutes < 60) {
    statusOffline = `был(а) ${diffMinutes} мин. назад`;
  } else if (diffMinutes < 1440) {
    const hours = Math.floor(diffMinutes / 60);
    statusOffline = `был(а) ${hours} ч. назад`;
  } else {
    const days = Math.floor(diffMinutes / 1440);
    statusOffline = `был(а) ${days} дн. назад`;
  }
}

// Показываем "в сети" только если статус online И пользователь был активен недавно
if (isOnline && isRecentlyActive) {
  statusOnline = 'в сети';
  statusOffline = 'в сети';
} else {
  statusOnline = statusOffline;
}
```

Теперь:
- ✅ "В сети" показывается только если пользователь был активен менее 2 минут назад
- ✅ Если пользователь вышел, статус меняется на "был(а) X мин. назад"
- ✅ Точное время последней активности отображается корректно

## Технические детали

### Логика определения онлайн статуса:
- Пользователь считается "в сети" если:
  - `userData.status === 'online'` (статус в базе данных)
  - И `lastSeen` был менее 2 минут назад
- Если хотя бы одно условие не выполняется, показывается время последней активности

### Обновление статуса:
- При открытии приложения: `status = 'online'`
- При закрытии вкладки: `status = 'offline'` (через `beforeunload`, `pagehide`)
- При смене видимости вкладки: `status = 'online'/'offline'`
- Каждые 30 секунд: обновление `lastSeen` если вкладка активна
