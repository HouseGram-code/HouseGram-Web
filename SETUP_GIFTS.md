# Настройка функции "Мои подарки"

## Шаг 1: Создание таблицы в Supabase

1. Откройте ваш проект в Supabase Dashboard
2. Перейдите в раздел **SQL Editor**
3. Создайте новый запрос
4. Скопируйте и выполните содержимое файла `supabase-received-gifts.sql`

## Шаг 2: Проверка таблицы

После выполнения SQL проверьте:
- Таблица `received_gifts` создана
- Индексы созданы
- RLS политики активны

## Шаг 3: Тестирование

1. Отправьте подарок другому пользователю через раздел "Молнии" → "Отправить подарок"
2. Зайдите под получателем
3. Откройте "Настройки" → "Мои подарки"
4. Подарок должен отображаться в списке
5. Нажмите "Обменять" для конвертации в молнии с комиссией 2%

## Структура таблицы

```
received_gifts
├── id (UUID) - уникальный идентификатор
├── user_id (TEXT) - ID получателя
├── gift_id (TEXT) - ID типа подарка
├── name (TEXT) - название подарка
├── emoji (TEXT) - эмодзи подарка
├── cost (INTEGER) - стоимость в молниях
├── from_user_id (TEXT) - ID отправителя
├── from_name (TEXT) - имя отправителя
├── can_convert (BOOLEAN) - можно ли обменять
└── received_at (TIMESTAMP) - дата получения
```

## Комиссия при обмене

При обмене подарка на молнии взимается комиссия 2%:
- Подарок стоимостью 15 ⚡ → получите 14 ⚡ (комиссия 1 ⚡)
- Подарок стоимостью 100 ⚡ → получите 98 ⚡ (комиссия 2 ⚡)

## Безопасность (RLS)

Политики Row Level Security обеспечивают:
- Пользователи видят только свои подарки
- Пользователи могут удалять только свои подарки
- Любой может отправлять подарки
