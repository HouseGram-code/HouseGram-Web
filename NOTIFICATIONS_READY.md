## Как протестировать

### 1. Разрешите уведомления

При первом входе браузер запросит разрешение:
- Нажмите **"Разрешить"** когда появится запрос

### 2. Проверьте что уведомления работают

1. Откройте приложение в двух разных браузерах (или в обычном и инкогнито)
2. Войдите под разными аккаунтами
3. Отправьте сообщение с первого аккаунта
4. На втором должно прийти уведомление!

### 3. Проверьте фоновые уведомления

1. Закройте вкладку с приложением (или сверните браузер)
2. Отправьте сообщение с другого устройства
3. Должно прийти системное уведомление Windows

## Что происходит при отправке сообщения

1. Сообщение сохраняется в Firestore
2. Получается FCM токен получателя из базы данных
3. Отправляется запрос на `/api/send-notification`
4. Firebase Cloud Messaging доставляет уведомление
5. Получатель видит уведомление (даже если браузер закрыт!)

## Особенности

- 🔔 Уведомления приходят даже когда браузер закрыт
- 📳 Вибрация при получении уведомления
- 🔊 Звук уведомления (если включен в настройках)
- 👆 Клик по уведомлению открывает приложение
- ⚡ Мгновенная доставка через Firebase

## Устранение неполадок

### Уведомления не приходят

1. **Проверьте разрешения:**
   - Windows: Параметры → Система → Уведомления
   - Chrome: Настройки → Конфиденциальность → Уведомления

2. **Проверьте консоль браузера (F12):**
   - Должно быть сообщение "FCM Token: ..."
   - Не должно быть ошибок

3. **Проверьте Service Worker:**
   - DevTools → Application → Service Workers
   - Должен быть `firebase-messaging-sw.js` со статусом "activated"

### Service Worker не регистрируется

1. Очистите кэш: DevTools → Application → Clear storage
2. Перезагрузите страницу (Ctrl+F5)
3. Проверьте что используете HTTPS или localhost
