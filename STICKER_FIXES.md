## Технические преимущества

### Почему Canvas API быстрее библиотек:
1. Нативная реализация браузера (написана на C++)
2. Нет overhead от JavaScript библиотек
3. Прямой доступ к GPU для ускорения
4. Минимальное использование памяти
5. Мгновенная обработка изображений

### WebP vs PNG:
- WebP на 30-50% меньше размер
- Поддерживается всеми современными браузерами
- Лучшее сжатие с сохранением качества
- Быстрая загрузка и отображение

## Как настроить Firebase Storage

### Шаг 1: Разверните правила Storage
```bash
firebase deploy --only storage
```

Или вручную скопируйте содержимое `storage.rules` в Firebase Console → Storage → Rules

### Шаг 2: Проверьте работу
1. Откройте http://localhost:3000
2. Войдите в систему
3. Откройте чат
4. Нажмите 🙂 → Стикеры → кнопка "+"
5. Загрузите стикер (обработка займёт < 100ms!)

## Как использовать свои стикеры

### Создание стикера:
1. В чате нажмите на иконку смайлика (🙂)
2. Перейдите во вкладку "Стикеры"
3. Нажмите кнопку "+" (плюс)
4. Выберите изображение (любой формат: PNG/JPG/WEBP/GIF)
5. Изображение мгновенно сжимается до WebP 512x512
6. Введите название стикера в модальном окне
7. Нажмите "Создать"
8. Стикер загружается за секунды!

### Просмотр моих стикеров:
1. В чате нажмите на иконку смайлика (🙂)
2. Перейдите во вкладку "Мои стикеры"
3. Здесь отображаются все ваши сохранённые стикеры

### Отправка стикера:
1. Нажмите на стикер в панели стикеров
2. Стикеры автоматически отправляются в чат

### Удаление стикера:
1. Перейдите во вкладку "Мои стикеры"
2. Нажмите кнопку "-" на стикере

## Структура хранения

Стикеры хранятся в Firebase Storage по пути:
```
stickers/{userId}/{timestamp}_{stickerName}.webp
```

Ссылки на стикеры сохраняются в localStorage браузера:
```
housegram_stickers: ["url1", "url2", ...]
```
