# 🚀 Инструкция по запуску Supabase Storage

## Что нужно сделать

### 1️⃣ Откройте Supabase Dashboard
Перейдите на: https://app.supabase.com

### 2️⃣ Выберите ваш проект
Кликните на проект HouseGram

### 3️⃣ Откройте SQL Editor
В левом меню найдите **SQL Editor** и кликните на него

### 4️⃣ Создайте новый запрос
Нажмите кнопку **New Query**

### 5️⃣ Скопируйте SQL код
Откройте файл `supabase-storage-setup.sql` и скопируйте **ВЕСЬ** код (Ctrl+A, Ctrl+C)

### 6️⃣ Вставьте и запустите
1. Вставьте код в SQL Editor (Ctrl+V)
2. Нажмите кнопку **Run** (или Ctrl+Enter)
3. Подождите 5-10 секунд

### 7️⃣ Проверьте результат
Вы должны увидеть сообщение:
```
============================================
Supabase Storage успешно настроен!
============================================
```

### 8️⃣ Проверьте Storage
1. В левом меню кликните **Storage**
2. Вы должны увидеть bucket с именем **files**
3. Статус должен быть **Public** (зелёная галочка)

## 🔧 Полезные команды

После настройки вы можете запускать эти команды в SQL Editor:

### Проверить размер хранилища
```sql
SELECT * FROM get_bucket_size('files');
```

### Статистика по пользователям
```sql
SELECT * FROM get_user_storage_stats() LIMIT 10;
```

### Топ-10 самых больших файлов
```sql
SELECT 
  name,
  pg_size_pretty((metadata->>'size')::bigint) as size,
  created_at
FROM storage.objects
WHERE bucket_id = 'files'
ORDER BY (metadata->>'size')::bigint DESC
LIMIT 10;
```

### Очистить файлы старше 90 дней
```sql
SELECT * FROM cleanup_old_files(90);
```

## ❓ Если что-то пошло не так

### Ошибка "relation users does not exist"
Сначала запустите основную схему БД из файла `supabase-schema.sql`

### Bucket не создаётся
1. Перейдите в Storage
2. Если bucket `files` уже существует - удалите его
3. Запустите SQL скрипт снова

### Ошибка прав доступа
Убедитесь, что вы запускаете SQL в Supabase Dashboard (там автоматически права администратора)
