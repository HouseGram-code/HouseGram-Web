# Исправление проблемы спама обновлений подарков

## Проблема
Подарки и сообщения постоянно обновлялись/спамились как в чате, так и на главной странице.

## Причины

### 1. Бесконечный цикл в ChatContext.tsx (КРИТИЧНО)
**Файл:** `context/ChatContext.tsx`

**Проблема:** В useEffect для подписки на сообщения была зависимость от `contacts`:
```typescript
}, [activeChatId, user, contacts]); // ❌ НЕПРАВИЛЬНО
```

Это создавало бесконечный цикл:
1. `onSnapshot` получает новое сообщение
2. `setContacts` обновляет состояние `contacts`
3. `useEffect` срабатывает снова из-за изменения `contacts`
4. Создается новая подписка
5. Цикл повторяется бесконечно

**Исправление:**
```typescript
}, [activeChatId, user]); // ✅ ПРАВИЛЬНО
```

### 2. Бесконечный цикл в подписке на статусы пользователей
**Файл:** `context/ChatContext.tsx`

**Проблема:** Зависимость от `Object.keys(contacts).join(',')` пересоздавалась при каждом изменении `contacts`:
```typescript
}, [user, Object.keys(contacts).join(',')]); // ❌ НЕПРАВИЛЬНО
```

**Исправление:** Использование `useMemo` для стабильного списка ID:
```typescript
const contactIds = useMemo(() => {
  return Object.keys(contacts).filter(id => 
    id !== 'saved_messages' && id !== 'test_bot'
  );
}, [Object.keys(contacts).sort().join(',')]);

useEffect(() => {
  // ...
}, [user, contactIds]); // ✅ ПРАВИЛЬНО
```

### 3. Избыточные подписки в SendGiftView.tsx
**Файл:** `components/SendGiftView.tsx`

**Проблема:** Подписка на количество подарков в реальном времени вызывала обновления у ВСЕХ пользователей при каждой отправке подарка:
```typescript
const unsubEaster = subscribeToGiftCount('easter_bunny', (count) => {
  setEasterBunnyRemaining(Math.max(0, 15 - count));
}); // ❌ Спам обновлений
```

**Исправление:** Загрузка данных один раз при монтировании:
```typescript
useEffect(() => {
  getGiftCount('easter_bunny').then(sent => {
    setEasterBunnyRemaining(Math.max(0, 15 - sent));
  });
  // НЕ подписываемся на изменения в реальном времени
}, []); // ✅ Загрузка один раз
```

## Внесенные изменения

### context/ChatContext.tsx
1. ✅ Добавлен импорт `useMemo`
2. ✅ Удалена зависимость `contacts` из useEffect для подписки на сообщения
3. ✅ Добавлен `useMemo` для стабильного списка `contactIds`
4. ✅ Обновлены зависимости useEffect для подписки на статусы пользователей

### components/SendGiftView.tsx
1. ✅ Удалены подписки `subscribeToGiftCount` в реальном времени
2. ✅ Оставлена только загрузка данных при монтировании компонента
3. ✅ Подписка на баланс звезд оставлена (нужна для актуального отображения)

## Результат
- ✅ Подарки больше не обновляются постоянно
- ✅ Сообщения в чате не спамятся
- ✅ Главная страница работает стабильно
- ✅ Производительность значительно улучшена
- ✅ Нет лишних запросов к Firebase

## Тестирование
Проверьте следующие сценарии:
1. Отправка подарка в чате - должен появиться один раз
2. Открытие чата - сообщения загружаются один раз
3. Открытие страницы отправки подарков - данные загружаются один раз
4. Баланс звезд обновляется только при реальном изменении

## Дополнительные рекомендации
1. Всегда проверяйте зависимости в useEffect
2. Избегайте подписок на глобальные коллекции в реальном времени
3. Используйте `useMemo` для стабильных значений
4. Проверяйте, действительно ли нужна подписка в реальном времени
